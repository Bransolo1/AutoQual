# Privacy Overview

This document provides a high-level overview of how the platform handles data. It is not legal advice.

## Data processed
The platform may process:
- User identifiers for authentication and authorization
- Study metadata and research artifacts (transcripts, clips, insights)
- Operational logs and audit events

## Data usage
Data is used to:
- Run moderated research sessions
- Generate analysis and evidence-backed insights
- Provide auditability and operational monitoring

## Retention
Retention is configurable at the workspace level. Operators should define retention policies
appropriate for their compliance requirements.

## Security
Data access is controlled by role-based access and workspace isolation. Audit events are recorded
to provide traceability for sensitive actions.

## Deletion and export
Operators can export audit logs and enforce retention policies. Additional deletion workflows
can be implemented per organization policy.
