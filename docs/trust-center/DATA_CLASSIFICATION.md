# Data Classification Policy (Template)

## Levels
- Public: approved for public release
- Internal: non-public business information
- Confidential: customer data and research artifacts
- Restricted: authentication data, secrets, and audit exports

## Handling requirements
- Confidential/Restricted data requires encryption in transit
- Restricted data requires least-privilege access and audit logging
- External sharing requires approval and access controls

## Examples
- Public: marketing content, docs
- Internal: product roadmaps, internal metrics
- Confidential: transcripts, clips, insights
- Restricted: access tokens, secrets, audit exports
